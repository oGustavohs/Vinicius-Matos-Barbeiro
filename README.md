# Vinicius Matos Barber

Primeiro projeto para cliente real. O Vinicius é barbeiro em Joinville e precisava de uma presença online que tivesse a mesma identidade do trabalho dele — preciso, com personalidade, sem parecer template.

O resultado foi um site institucional completo, entregue e no ar.

**[→ Ver o site](https://ogustavohs.github.io/Vinicius-Matos-Barbeiro/)**

---

## O que tem no site

Não é um site vitrine estático. Tem sistema de agendamento com calendário interativo, integração com WhatsApp, tabela de serviços e preços, galeria de trabalhos, sistema de avaliações com estrelas e um layout que funciona bem no celular — que é de onde a maioria dos clientes acessa.

O fluxo de agendamento funciona assim: o cliente escolhe o serviço, seleciona o dia no calendário, escolhe o horário disponível e confirma direto pelo WhatsApp com os dados preenchidos automaticamente.

---

## Stack

```
index.html   → estrutura e componentes
style.css    → identidade visual, responsividade, animações
script.js    → calendário, agendamento, sistema de avaliações
```

HTML · CSS · JavaScript puro

---

## O que esse projeto me ensinou

Desenvolver para um cliente real é diferente de desenvolver pra si mesmo. Você precisa pensar no usuário final que nunca vai ver o código — só vai ver se o botão funciona ou não. Cada detalhe de UX virou uma decisão consciente.

Também foi a primeira vez que construí um sistema de calendário do zero, sem biblioteca. Mais trabalhoso, mas aprendi muito mais do que se tivesse usado um plugin pronto.
